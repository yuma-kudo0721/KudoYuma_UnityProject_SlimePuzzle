# スライム分離・合体パズルアクション（仮）

## 概要
本作は、**スライムの分離・合体を活かしたパズルアクションゲーム**として制作している作品です。  
当初は、分離・合体という1つの仕組みから複数の解法が生まれるパズルゲームを目指して企画しました。
現在はそのコアとなる、
- プレイヤー操作
- スライムの分離・合体
- 状態切り替え（通常／蝙蝠モード）
といったシステム部分を中心に実装しています。

## ゲームの特徴
- スライムを弾として発射し、分離できる
- 分離したスライムは回収・合体が可能
- 合体量によってサイズ・重さが変化
- 通常状態と能力を切り替えて攻略
- アクションとパズルの両立を意識した設計

## 操作方法
- ← → ：移動  
- Z ：ジャンプ  
- Space ：スライムを発射（分離）  
- X ：分離したスライムを実体化  
- C ：操作スライムの切り替え  

## 開発環境
- Unity  
- C#  
- Visual Studio Code  

## プログラム面のこだわり
### 分離・合体システム
- 本体および分離したスライムをList<GameObject>で管理
- 合体量をslimeSizeとして数値管理し、
  - キャラクターサイズ
  - 発射可能回数
  - Rigidbody2D の質量  
  に反映
分離合体を駆使して攻略できるように考えてます。

### 操作キャラクターの切り替え
複数のスライムが存在しても、操作対象は常に1体のみとしています。

切り替え時には、
- 位置
- サイズ
- 管理リスト上のインデックス
を入れ替える方式を採用し、  
入力処理やカメラ制御を複雑にしない構造にしました。

### 状態切り替え（通常／蝙蝠モード）
Animator Controller を実行時に切り替えることで、  
同一キャラクターに複数の行動特性を持たせています。
蝙蝠モードではジャンプ回数が増加するなど、  
アクションの幅を広げています。

### 設計で意識した点
- 1つの仕組み（分離・合体）から複数の攻略方法が生まれること
- 後からギミックやステージを追加しやすい構造にすること
